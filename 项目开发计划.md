# AnimeRenamer v2 — 开发计划

> 自动化动画整理系统：qBittorrent → 智能重命名 → NAS 存储 → Jellyfin 媒体库

---

## 🎯 项目目标

填补 NAS 追番工作流的最后一块拼图：

```
qBittorrent 下载完成
        ↓
  [现在是手动操作]  ← 我们要自动化这里
        ↓
  Jellyfin 识别播放
```

**目标状态：**
- 追番：qBittorrent 下载完成后，**零操作**进入 Jellyfin 追番库
- 补番：自动识别 + Web UI **一次点击**确认后进入 Jellyfin 收藏库
- 所有操作通过 Web Dashboard 可视化管理

## 🔀 双模式工作流

### Mode 1: `auto` — 追番（当季新番）

```
下载完成事件
    → 解析文件名
    → TMDB 验证（置信度 > 阈值）
    → 查 series_config.yaml → mode: auto ✅
    → rename → move → hardlink
    → 写操作日志
    → 触发 Jellyfin 刷新（可选）
    → 完成（零人工干预）
```

**失败 fallback：** 任何步骤出错 → 停止执行 → 进入 Web UI 待确认队列，**绝不执行有风险的操作**

### Mode 2: `confirm` — 补番 / 未知番剧

```
下载完成事件 / 手动扫描
    → 解析文件名
    → TMDB 匹配 + 置信度评分
    → 进入 Web UI "待处理" 队列
    → 用户在 Dashboard 看到预览：
        📁 Mushoku Tensei II - 13.mkv
        识别结果：无职转生 第二季
        TMDB ID：94664 ✅
        目标路径：/anime/无职转生/Season 2/无职转生 S02E13.mkv
        [✅ 确认] [✏️ 修改] [⏭️ 跳过]
    → 用户确认 → 执行
```

## ⚙️ series_config.yaml 规范

```yaml
# 全局设置
settings:
  tmdb_api_key: "${TMDB_API_KEY}"       # 通过环境变量注入
  confidence_threshold: 0.85            # TMDB 匹配置信度阈值
  default_mode: confirm                 # 保守策略：未配置番剧默认需确认

  # 路径配置（容器内路径，通过 volume 映射到 NAS 实际路径）
  download_dir: "/downloads"
  storage_dir: "/anime"
  jellyfin_airing_dir: "/jellyfin/追番"     # auto 模式番剧的 hardlink 目标
  jellyfin_collect_dir: "/jellyfin/动漫"    # confirm 模式番剧的 hardlink 目标

# 番剧配置
series:

  # 追番示例（全自动）
  "Dungeon Meshi":
    mode: auto
    tmdb_name: "Delicious in Dungeon"
    tmdb_id: 114410
    season: 1
    aliases:                 # 文件名可能出现的变体，辅助解析器匹配
      - "Dungeon Meshi"
      - "迷宫饭"
      - "Delicious in Dungeon"

  # 补番示例（手动确认）
  "Mushoku Tensei":
    mode: confirm
    tmdb_name: "Mushoku Tensei: Jobless Reincarnation"
    tmdb_id: 94664
    seasons:
      - { season: 1, status: done }
      - { season: 2, status: in_progress }
    aliases:
      - "Mushoku Tensei"
      - "无职转生"
      - "Jobless Reincarnation"
```

## 🧩 文件名解析器支持的字幕组格式

解析器需要覆盖以下主流格式：

| 字幕组 | 示例文件名 |
|---|---|
| SubsPlease | `[SubsPlease] Dungeon Meshi - 15 (1080p) [ABC123].mkv` |
| ANi | `[ANi] Re:从零 第三季 - 01 [WebRip 1080p AVC AAC][CHT].mp4` |
| Lilith-Raws | `[Lilith-Raws] Mushoku Tensei II - 13 (WebRip 1080p HEVC).mkv` |
| 极影字幕社 | `【极影字幕社】★04月新番 忘却Battery 第02话 GB 1080p.mp4` |
| Sakurato | `[Sakurato] Spice and Wolf - 03 [AVC-8bit 1080p AAC][CHS].mp4` |
| Mikan Project | `[Mikan Project] Slime 3rd Season - 02 [WebRip 1080p HEVC].mkv` |
| 简化命名 | `05.mkv`（需要配合 torrent 名称识别） |
| 自定义 | 通过 `aliases` 配置兜底 |

## 🐳 Docker 部署规范

```yaml
# docker-compose.yml 草稿
version: "3.8"
services:
  anime-triage:
    build: .
    container_name: anime-triage
    ports:
      - "8765:8765"       # Web UI 端口（待定）
    environment:
      - PUID=1000          # ← 用户提供 Unraid UID
      - PGID=100           # ← 用户提供 Unraid GID
      - TMDB_API_KEY=xxx   # ← 用户提供
    volumes:
      - /mnt/user/downloads:/downloads              # qBt 下载目录
      - /mnt/user/hentaidisk/video/anime:/anime     # 存放目录（待确认）
      - /mnt/user/hentaidisk/video/link:/jellyfin   # Jellyfin 媒体库（待确认）
      - ./config:/app/config                        # 持久化配置
      - ./logs:/app/logs                            # 持久化日志
    restart: unless-stopped
```

**权限方案：** 容器以 PUID/PGID 运行，写出的文件权限自动匹配 Unraid 用户，`change_own.sh` 可以退休。

## 🤖 Agent 分工

### Agent A — 后端核心（Parser + TMDB + Triage 逻辑）
**负责：**
- `parser.py`：正则 + 规则引擎，解析各字幕组文件名
- `tmdb.py`：TMDB API 封装，搜索/验证/置信度评分
- `triage.py`：rename、move、hardlink、rollback 核心操作
- `models.py`：所有数据结构定义

**输出标准：** 每个函数有单元测试，在没有真实目录的情况下可以 dry-run

---

### Agent B — API 层（FastAPI 后端）
**负责：**
- `main.py`：RESTful API endpoints
- `watcher.py`：目录监控守护进程（watchdog）
- 与 qBittorrent webhook/API 的集成
- 操作日志记录

**核心 API 设计：**
```
GET  /api/queue               → 待处理列表
GET  /api/history             → 操作历史
POST /api/queue/{id}/confirm  → 确认执行
POST /api/queue/{id}/skip     → 跳过
POST /api/scan                → 手动触发扫描
GET  /api/series              → 获取所有番剧配置
POST /api/series              → 新增番剧配置
PUT  /api/series/{name}       → 更新番剧配置
```

---

### Agent C — Web UI（Frontend Dashboard）
**负责：**
- Dashboard 主页：待处理队列 + 最近操作历史
- 番剧配置管理页：查看/添加/编辑 series_config
- 操作日志页
- 设置页：全局参数配置

**设计要求：**
- 深色主题，现代风格
- 响应式（手机/平板/桌面）
- 待确认卡片要清晰显示：原文件名、识别结果、TMDB 封面、目标路径预览

---

### Agent D — DevOps（Docker + Unraid 部署）
**负责：**
- `Dockerfile` 与 `docker-compose.yml`
- PUID/PGID 权限处理
- Volume 挂载规划
- 健康检查与自动重启策略
- Unraid Community Applications 模板（可选）

## 🔒 安全原则

1. **保守优先**：置信度低于阈值 → 一律进入 confirm 队列
2. **干运行模式**：所有操作支持 `--dry-run`，只打印不执行
3. **操作日志**：每次 rename/move/hardlink 都有完整记录，方便回滚
4. **Hardlink 安全**：hardlink 失败不影响原文件；原文件在确认 hardlink 成功前不删除
5. **权限隔离**：以最小权限运行，PUID/PGID 匹配 NAS 用户
